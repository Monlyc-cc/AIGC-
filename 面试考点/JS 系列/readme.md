# 3. 谈谈 js 中的类型转换机制
- 是什么
   - 隐式类型转换： 不同的类型间的数据进行计算或者判断是，js引擎会自动将一种数据类型转换为另一种数据类型，从而进行计算
    - 显式类型转换： 人为的将一种类型转换成另一种数据类型，
- 特点
  - 显式类型转换的方法： 调用已存在方法
  - 隐式类型转换： 通常发生在四则运算判断语句、if、while，==场景中， 当运算符左右两边的数据类型不同时，会转为相同类型，再进行计算。 
        1. if，while 转换为布尔值
        2. 四则运算： 有一方为字符串，则另一方也转换为字符串，进行字符串拼接。 否则，两方都转换为数字，进行数字运算。

- 场景
    - 比较操作符
    - 逻辑操作符
    - 函数调用
    - 模板字符串

            - 引用类型 ： 而拷贝主要发生在引用类型上
        而拷贝的方式主要分为两种方式
            - 浅拷贝，顾名思义浅层的拷贝，当被拷贝的引用类型内部存在引用类型，浅拷贝只会将这份引用地址拷贝下来，而不会拷贝出一个一样的新的引用类型对象

# 6. 聊一聊js中的闭包
    - 是什么
        调用一个外部函数中返回的内部函数时，即使外部函数已经执行结束，但是内部函数依然引用了外部函数中的变量，那么外部函数的词法环境将被保留存进堆中，而剩下的外部词法环境与内部函数的组合，我们称之为闭包。
    - 特点
         1. 内存泄漏
         2. 定义私有变量，防止全局污染
         3. 延长变量的生命周期

    - 应用场景
         1.封装私有变量
         2.模块化

# 9. 聊聊 js 的原型
- 是什么
    1. js函数有二义性，除了直接执行，也可以配合 new 关键字 创建实例对象
    2. 每一个函数都拥有一个prototype属性，这个属性指向一个对象，我们称之为显示原型对象。
    3. 每一个由函数创建的实例对象，都有一个__proto__属性，这个属性指向函数的显示原型对象。
    4. 由此每一个实例对象都可以访问到显式原型对象中的属性和方法
- 特点
    1. 实例对象可以访问到显示原型对象中的属性和方法
    2. 显示原型对象可以访问到构造函数中的属性和方法
- 应用场景
    1. 实现继承
    2. 实现封装
- 特例
    1. 构造函数的prototype属性指向的对象，默认都有一个constructor属性，指向构造函数本身。

# 9. 聊一聊 js 中的继承
- 是什么
    1. 让子类可以访问到父类的属性和方法
- 特点
    1. 原型链继承 -- 让子类的显式原型 指向 父类的实例对象，从而实现继承。 缺点： 多个子类实例 依赖 同一份 父类实例，会相互影响
    2. 构造函数继承 -- 在子类中将父类的属性和方法绑定到子类的实例对象上。 缺点： 无法继承父类的原型方法
    3. 组合继承 -- 将前两种方法结合起来，在子类中将父类的属性和方法绑定到实例对象上，同时让子类的显式原型 指向父类的实例对象，从而实现继承。 缺点： 调用了两次父类的构造函数，生成了两份实例。， 并且这样的方法使得子类失去了自己的显式原型，失去了自义专属于子类自己共有属性与方法
    4. 寄生组合继承 --  是组合继承的一种优化方案，避免了调用两次父类构造函数
    5. 类继承
# 10.聊一聊 js 中的this
- 是什么
    1. this是函数执行上下文中的一个属性，它的值取决于函数的调用方式，是动态绑定的。
- 特点
    绑定规则：
    1. 默认绑定，函数独立调用，this指向全局对象 window
    2. 隐藏绑定, 当函数作为对象的方法调用时，this指向调用该方法的对象
        - 隐式丢失： 当函数作为对象的方法，并且是多层调用时，this指向离这个函数最近的对象
    3. 显式绑定 
        - call, apply, bind 方法可以显式的绑定 this 指向某个对象
        - call，apply绑定后立即执行
        - call，apply接收参数的方式不同，call接收多个参数，apply接收一个数组参数
        - bind 只绑定，不执行
- 应用场景
    1. 普通函数调用
    2. 构造函数调用
    3. 方法调用
    4. 箭头函数调用

# 11. 聊一聊 js 事件模型（事件流）
- 是什么 
    js 给容器绑定的事件的触发时分为三个阶段
    1. 事件捕获阶段， 事件从window开始，是由上而下传递，直到目标元素
    <!-- 2. 目标阶段， 事件到达目标元素 -->
    3. 事件冒泡阶段， 事件从目标元素开始，是由下而上触发，直到window

- addEventListener(‘事件名’，回调函数，是否在捕获阶段触发) true在捕获阶段触发，false在冒泡阶段触发
    - 事件的传递，是由上而下传递的
    - 事件的触发，是由下而上触发的
- 应用场景
    - 事件委托，当我们要为子容器绑定点击事件时，可以为父容器绑定一个委托事件，通过 e.target 访问到目标容器，减少事件绑定的次数。
    - 阻止事件冒泡 e.stopPropagation(), 在子组件事件触发时，阻止事件冒泡，不触发父组件的功能


# 12 . 说一说js中的类型判断
- 是什么：js中存在一些方法可以用来判断变量的类型。
    1. typeof 操作符可以判断基本数据类型（string, number, boolean, undefined, symbol）
        原理：是将数据转换为二进制，如果是 000 开头，就是对象类型，否则就是基本数据类型。
        注意： null 转换为二进制是全0，所以会被判断为对象类型，不能用 typeof 判断。
    2. instanceof 操作符可以判断引用数据类型（Array, Object, Function）
        原理： 是通过原型链来查找，通过判断左边对象的隐式原型，是否指向右边对象的显示原型。
        注意： 如果原型链被更改过，导致判断结果出错
    3. constructor 构造函数属性可以判断引用数据类型
    4. Object.prototype.toString.call() 方法可以判断所有数据类型
    5. Array.isArray() 方法可以判断数组类型
- 应用场景
    1. 判断变量的数据类型
    2. 类型转换

# 15. js中如何实现函数缓存
- 是什么
    1. 缓存结果
    2. 每次执行前，寻找是否有相同的执行记录，如果有则不进行计算，直接返回结果，否则进行计算，并留下记录
    将函数的执行结果缓存起来，当下次调用函数时，如果参数相同，就直接返回缓存的结果，而不是重新计算
- 应用场景
    1. 用于昂贵的函数调用
