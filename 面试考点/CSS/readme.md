# 1. css 盒子模型

回答问题三范式
- 是什么:
在浏览器渲染页面的时候，渲染会根据 css 盒子模型的标准来渲染容器，将每个容器分为内边距（padding）、边框（border）、外边距（margin）和内容（content）四个部分，进行渲染。

- 有什么优点
    - 盒子模型分为多种：
        - 标准盒子模型： css的width属性只包含 content的宽带
        - 怪异盒子模型： css属性中 width 属性是 content ，padding ，border共享的 

- 应用场景是什么
    - 盒子模型没有应用场景可言

# 3.说一说 em/px/rem/vh/vw 的区别
- px是绝对单位， 一个px就是一个像素点
- em  是父容器的字体大小
- rem 是根容器的字体大小
- vh  是视窗大小
- vw  是视窗宽度

# 4.css 中有哪些方式隐藏元素
更改容器类别
display: none;/* 隐藏元素，不占用文档流 ,直接从页面消失 ，不可触发事件*/

更改是否展示
visibility: hidden; /*隐藏元素,占用文档流,只是看不见，同时不可触发事件*/

更改透明度
opacity: 0; /*透明度设置，占据文档流 ，可以触发事件*/

利用缩放
transform: scale(0); /*缩放设置，占据文档流 ，可以绑定触发事件, 但是缩放到0时，因为容器过小无法点击，约定于不可触发*/

利用裁剪
clip-path: polygon(0% 0%, 0% 0%, 0% 0%, 0% 0%); /*裁剪容器 ， 占据文档流，裁剪为0时，不可触发事件*/           

利用定位            
position: absolute;/*利用定位将容器移到可视区域之外*/，不占据文档流
left: 9999999px;

# 6.谈一谈BFC容器
是什么？
- BFC（Block Formatting Context）是指块级格式化上下文，是一个独立的渲染区域，内部的元素渲染不会影响到外部的元素，外部的元素也不会影响到内部的元素。

特点：
- 内部元素的 margin 不会与外部元素的 margin 发生折叠
- BFC 容器计算高度时，会计算浮动元素高度
- BFC 容器不会与浮动元素重叠
- 内部元素，不影响外界

如何使用BFC容器
BFC (block formatting context)（块级格式化上下文）
- 普通容器中存在  父容器的 margin-top 与 子容器的margin- top 重叠的问题


- BFC是一个拥有特殊渲染规则的隔离空间

- 如何创建一个BFC容器
1. overflow: hidden | auto | scroll | overlay
2. position:absolute | fixed
3. float: left | right
4. display: inline-block | flex | grid

应用场景
1. 清楚浮动
2. 防止margin 与 子元素margin 重叠



# 7. 怎么理解回流重绘 （ 浏览器输入url后，发生了什么 ）

- 输入 url 后发生了什么？
1. 网络传输 
    -  进行DNS域名解析，得到目标ip地址
    -  向目标ip进行tcp连接
    -  根服务器将数据分为多个数据包，根据http协议向客户端传输
    1. 由此获得HTML,CSS,JS等资源

2. 浏览器渲染
- 浏览器获取到服务端的资源后，会做什么
1. 解析HTML资源，生成DOM树
2. 解析CSS资源，生成CSSOM树
3. 将DOM树与CSSOM树合并，生成渲染树
4. 计算页面布局(回流) ：回流的目的在于，重新计算页面布局，当页面元素的几何属性变化时，才会发生
5. 渲染页面（gpu）（重绘）

    - 发生回流的场景： 当元素的几何属性发生变化时，会触发回流，浏览器回重新计算元素的位置与大小
    - 发生重绘的场景： 当元素的非几何属性发生变化时，会触发重绘




    # offsetwidth 与 clientwidth
- offsetwidth ： 元素的宽度，包含内容，内边距，边框，不包含外边距
- clientwidth ： 元素的宽度，包含内容，内边距，不包含边框，不包含外边距

    # 浏览器的优化机制
-  优化队列：
    由于每一次回流都会造成额外的计算消耗，所以浏览器会维护一个优化队列，将会带来回流行为存放在队列中，直到一定时间后，或者达到队列阈值后，浏览器会一次性执行队列中的回流行为，根据最终结果进行浏览器的回流重绘

-  导致回流的特例： 强制属性的属性，这些数值在被获取时会清空一次回流优化队列，如果此时优化队列有值，则导致回流，否则不回流
    offsetwidth 与 offsetheight、...
    clientwidth 与 clientheight、...
    scrollwidth 与 scrollheight、...
    这些dom属性，是临时计算的，所以获取这些值时，会强制清空队列，从而到当前准确值，如果当前任务队列为空，则不带来回流。如果队列不为空，则导致回流触发

    # 应用场景：
  先将元素脱离文档流后，对其几何属性进行修改，修改完成后再插入文档流中，可以减少渲染时回流重绘的次数。

# 8.什么是响应式布局？我是怎么代码中实现响应式布局的？ 
 1. 什么响应式布局： 响应式布局是指，当页面根据设备尺寸，自动调整布局与样式，从而提升用户体验
 2. 实现响应式布局的方式：
    - 媒体查询： 利用css中的@media 查询用户屏幕尺寸，根据不同的屏幕尺寸，设置不同的css样式
    - 弹性布局flex： 利用flex布局，根据不同的屏幕尺寸，设置不同的flex布局规则
    - 网格布局grid： 利用grid布局，根据不同的屏幕尺寸，设置不同的grid布局规则
    使用相对单位：
    - 百分比布局： 利用百分比布局，根据不同的屏幕尺寸，设置不同的元素宽度
    - rem 单位 + 媒体查询 ， 根据屏幕尺寸更改rem值，可以实现响应式布局

# 9. css 中常见的实现动画的方式
1. 过渡动画 （transition）
    - 为属性的变化添加一个过渡时间
    - 过渡动画是指，当元素的属性发生变化时，会自动过渡到新的属性值，而不是瞬间变化。
    - 过渡动画的实现，需要设置元素的过渡属性，包括过渡属性，过渡时间，过渡函数等。
2. 自定义动画 （animation）
    - 动画动画是指，当元素的属性发生变化时，会自动过渡到新的属性值，而不是瞬间变化。
    - 动画动画的实现，需要设置元素的动画属性，包括动画属性，动画时间，动画函数等。
3. 转变动画 （transform）
    - 用来移动，转动dom元素
    - 转变动画是指，当元素的属性发生变化时，会自动过渡到新的属性值，而不是瞬间变化。
    - 转变动画的实现，需要设置元素的转变属性，包括转变属性，转变时间，转变函数等。
4. 使用动画库
    - animate.css ： 一个非常流行的动画库，提供了很多常用的动画效果，只需要引入css文件，即可使用
