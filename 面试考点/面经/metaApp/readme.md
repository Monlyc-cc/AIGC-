1. 有哪些方式会导致内存泄漏（常见内存泄漏场景）
    - 闭包
    - 未清理的定时器
    - 意外的全局变量，没用使用var，let，const 等关键字声明的变量，会自动挂在到window中
    - 未清理的dom引用： 当我们在js中操作dom时，会将dom元素存储在内存中， 如果我们没有及时清理，就会导致内存泄漏。
    - 事件监听器未移除： 当我们在js中添加事件监听器时，被监听的dom对象被移除，响应的事件监听器不会被移除，就会导致内存泄漏。
    - 循环引用： 当两个对象相互引用，且没有其他引用指向它们时，就会导致内存泄漏。
2. 什么是循环引用？
    - 循环引用是指两个或多个对象相互引用，形成一个循环，导致内存泄漏。
    - 例如，当一个对象A引用了对象B，而对象B引用了对象A时，就会形成循环引用。
    - 循环引用的问题在于，即使没有其他引用指向这两个对象，它们也会一直存在于内存中，因为它们之间的引用关系会阻止垃圾回收机制对它们进行回收。
3. addEventLinstenr() 接受多少个参数？ 分别用什么作用？
    - 接受3个参数： 事件类型，事件处理函数， 是否在捕获阶段触发（可选）
    - 第一个参数传递，要绑定的事件类型
    - 第二个参数传递，要被出发事件函数
    - 第三个参数，决定事件是否在捕获阶段就触发

4. 事件捕获机制，冒泡机制，事件委托机制
    - 事件捕获机制： 事件的捕获从最外层的元素开始，依次向下传递，直到到达目标元素。
    - 事件冒泡机制： 事件的触发从目标元素开始，依次向上传递，直到到达最外层的元素。
    - 事件委托机制： 利用事件冒泡机制，将事件处理函数绑定在父元素上，当子元素触发事件时，会冒泡到父元素上，从而触发事件处理函数。
5. 箭头函数与普通函数的不同？
    - 箭头函数拥有自己的this， 普通函数没有
    - 箭头函数有自己的函数名，普通函数没有
    - 箭头函数不可以 new 构造实例对象，普通函数可以
    - 箭头函数没有arguments 对象，普通函数有
    - 箭头函数没有prototype 属性，普通函数有
    - 箭头函数没有自己的构造函数，普通函数有
    - 箭头函数可以简写，普通函数不行
    。。。。。
    反正箭头函数除了可以跟数组一样接受参数并执行内部逻辑，普通函数有的它几乎都没有，优点就是存在可以简单的定义一个临时函数，以及利用没有自身this的特性用闭包实现一些特殊功能，存在简写语法，比普通函数更加轻便。

5. new 做了什么？
    - 构建一个新对象
    - 将函数中的this显式绑定到这个新对象上
    - 执行函数体代码
    - 返回这个新对象：  存在特例，如果函数中返回了引用类型对象，那么new 后 永远返回的是指定的引用类型，而不是新构建的实例对象
6. 为什么箭头函数不可以 new
    - 箭头函数没有自己的this：new 操作符会创建一个新的对象，但是箭头函数没有自己的this， 所以不能用new 调用箭头函数。
    - 箭头函数没有prototype 属性： 因为箭头函数没有自己的this， 所以也没有prototype
