# 组件通信方式
1. 父向子通信的方式
    - 插槽
    - 在子组件标签中 写属性与属性值， 子组件可以通过props对象接受
2. 子向父传值
    - 父组件 提前 向子组件传一个状态更新函数，或者事件处理函数，子组件通过调用这个函数，将数据作为参数传递给父组件，父组件在函数中更新状态或执行其他操作。
    - 子组件使用、
        - forwardRef 来转发 ref 到子组件的 DOM 元素或子组件实例。这在需要在父组件中直接操作子组件的 DOM 元素或实例时非常有用。
        - useImperativeHandle 可以用来定义子组件暴露给父组件的方法。
3. 兄弟组件
    - useContext 来实现兄弟组件之间的通信。通过创建一个上下文（Context），可以在组件树中共享状态和方法。


# 函数组件与类组件的区别
1. 函数组件是一种简单的组件，它只接收 props 作为参数，并返回一个 React 元素。函数组件没有自己的状态（state），也没有生命周期方法。
2. 类组件是一种更复杂的组件，它继承自 React.Component 类，并且可以拥有自己的状态（state）和生命周期方法。类组件可以使用 this.state 来管理状态，并且可以在生命周期方法中执行一些操作，如组件挂载、更新和卸载。
3. 函数组件更简洁、更易于理解和维护，而类组件更灵活、更强大。在 React 中，函数组件通常用于展示性的组件，而类组件则用于需要状态管理和生命周期方法的组件。
