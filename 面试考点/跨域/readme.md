# 同源策略
- 浏览器自带的一种安全机制，限制不同源之间的交互，防止恶意网站窃取的数据

1. 什么是同源
    - 网络地址可以分为四个部分：协议，域名/ip，端口，路径
     https://192.168.1.10:8080/home 协议：https ，ip：192.168.1.10 ，端口：8080 ，路径：/home
    - 只有协议，ip，端口都相同时，前后端才同源
2. 特点
    - 世界上不存在同源的前后端，所以前端无法直接接受后端的响应
    - 所以需要主动去解决跨域
    - 
3. 解决跨域的方案
- 前端解决
    1. jsonp ，script 标签，img标签的对外请求不受同源策略约束,通过动态创建script标签来加载跨域资源
        - 缺点：
            1. 只支持get，不支持post等其他请求，并且get请求能携带的信息太少了
            2. 要求前后端配合，步骤繁琐
    2. cors ，服务器端设置响应头，指定允许跨‘’域的白名单
    3. 代理: 用node.js创建一个自己的服务器，然后向目标后端发送请求获取数据，由自己的后端对前端解决跨域问题
        - VITE ，在vite.config.js中配置代理,只能在生产环境下奏效
        - 
- 不使用http协议，同源策略只约束http协议
    4. websocket：利用websocket协议，实现全双工通信，避免同源策略限制，双方可以主动向对方发消息
       全双工，服务端可以主动向客户端推送消息
    5. postMessage：可以在不同源的窗口之间传递消息
        -  缺点： 1.只能在致辞html5的浏览器中使用   
    6.domain：利用document.domain 来实现跨域通信
        -  缺点： 1.只能在二级域名相同的情况下使用
