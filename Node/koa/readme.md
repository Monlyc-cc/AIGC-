# node 框架
- node 不是一个语言，是js的执行环境
在node开发中，一些模块的使用和配置，需求的处理比较复杂，为了提高开发效率，减少重复的工作，我们可以使用一些框架来帮助我们完成这些工作。




# 封装一个后端接口会用到的模块
1. 前提
    - 前后端通信必须符合http协议的规则，http协议是前后端交流的基石
2. 原生js内置的“http”模块
    - http模块，在符合http协议的前提下，可以定义一个后端接口
    - 缺点： 方式过于原始，后端逻辑处理过于繁琐


3. 非原生的模块 
存在公司或个人封装了基于http协议，构建后端接口的模块，比传统的使用http模块使用方便，功能更完善
- express 框架 ： 功能完整，但是太重
- koa 框架： 功能简单，但是灵活轻便。
- next 框架： 适用于超大型项目开发
