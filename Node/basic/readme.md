# node 
  什么是node？
  - node是js的一种运行环境：js原先只是单纯面向浏览器页面的设计语言，没有封装访问操作系统的能力（无法进行文件管理，无法创建一个后端服务，无法连数据库等等，没有办法与操作系统底层进行交互）
  为什么需要node？
  - node在保留v8引擎的功能前提下，封装了更多与计算机底层交互的代码为内置模块，由此使得js语言不再局限于浏览器页面，拥有与操作系统底层交互的能力。
     1. http模块：依据http协议构建一个后端接口
        - 接口内部设计简单的服务逻辑
        - 将后端接口逻辑跑在本地端口
        - 依据后端服务逻辑，分析用户的请求头传来的信息，向客户端传回数据（数据必须是json字符串）
    2. https模块： 功能与http基本相同
        - 不同点： 安全性强,后端向前端传输数据时，会对数据进行加密
    3. fs模块： 用于文件管理
        - 增删改查文件
        - 移动文件位置等等
## 模块封装的标准
1. 旧的标准： CommonJS 规范 
    - 该规范起源于node引擎，因为在js中封装模块的方法由node首创，使得js语言功能逐渐完善
    - 使用方法： 
2. 新的标准： ESmodule 规范， js官方定义的规范，起源于es6
    - 该规范诞生后使得js不再依赖于node引擎，就可以封装模块
ps： 
- 因为commonjs规范下封装了许多经典好用的模块，无法舍弃，现在项目依然有许多使用的是commonjs规范
