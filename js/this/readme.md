# 为什么要有 this ？
- this 提供了一种更优雅的方式来隐式的传递一个对象引用，可以让代码更简洁，易于复用

# this 通常用在哪儿？
1. 全局作用域   this == window
2. 函数作用域

- this 是一个代词，用在不同的地方代表不同的值。

# this 的绑定规则
1. 默认绑定 --- 当函数被独立调用时，函数中的this指向window。
2. 隐式绑定 --- 当函数被上下文对象引用，且被该对象调用时，函数中的this会绑定到上下文对象上。
3. 隐式丢失 --- 当一个函数被多层对象调用时，函数的this指向离函数最近的对象，其它的对象都丢失了 this 绑定。
4. 显式绑定 
- fn.call(obj,x,x) 显式的将fn里面的this绑定到obj对象上， call负责帮组fn接收参数。
- fn.apply(obj,[x,x]) 作用与call无异，只是apply接收参数的方式不同，apply接收的参数是一个数组。
- fn.bind(obj),显式将fn里面this绑定到obj对象上，可以只绑定但不触发函数


# new 绑定
- new 会导致函数内部的this指向实例对象。
- 当构造函数中存在return 并且，返回到是引用数据类型，则new的返回失效。

# 箭头函数 
- 箭头函数内部无法拥有this属性，写在箭头函数中的this是从外层非箭头函数下找到的。
