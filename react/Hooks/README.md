# 在 React 16.8 之前
- 类组件
1. 通过 this。state={} 来存放组件的状态  {状态指的是，一经修改就会导致组件重新渲染的变量}
2. 通过setState 来修改某个变量的状态，带来组件的重新渲染。

- 生命周期
旧版本react中设置了以下生命周期函数
1. constructor()//构造函数 ， 组件实例被创造时被调用，此时这个实例拥有生命周期了
2. render()//渲染函数，这个实例会被立即渲染，render函数就会被调用一次
3. componentDidMount()//加载函数，第一次渲染完毕时，会自动调用该函数
4. componentDidUpdate()//更新函数 ，每一次组件实例状态改变并渲染后会调用
5. componentWillUnmount()//卸载函数，组件实例被销毁前调用

- 函数组件
1. hooks 为函数滞组件带来了活力
    - useSate() ： 用来设置函数组件的状态，并且提供修改状态的函数，当组件状态被修改，则会导致组件重新渲染（具体过程，旧组件被卸载，新组建被渲染）
    - useEffect() 为函数组件定义副作用函数,被使用在函数组件中，其内部可以传入一个函数，而useEffect可以根据组件状态改变情况来决定是否执行内部函数
        - useEffect(()=>{}) 会在组件初次加载与每次状态更新时触发
        - useEffect(()=>{},[]) 只会在组件初次加载时触发
        - useEffect(()=>{},[count]) 会在组件初次加载与每次count状态改变时触发
            - 可以在[ ] 内部可以设置多个状态，只要其中任意一个状态改变，就会触发副作用函数
        - useEffect(()=>{},[])之中可以返回一个函数，这个函数会在页面卸载调用
    - ps: 组件每一次重新渲染 == 旧状态组件被卸载，新的组件产生，重新执行一遍组件内部函数，而被useEffect（）内部的函数会根据组件改变状态决定是否执行
