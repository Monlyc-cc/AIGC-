# 跨域

协议         域名     端口  路径

- 浏览器的同源策略：
1. 协议    域名    端口都相同的两端 才可以进行网络通信
2. 前端请求发出去，后端响应返回了，但是浏览器不接收


- 解决跨域
- 同源策略 使得 自己写的前端也无法接收到后端发来的数据，所以需要解决跨域问题
1. cors：（设置响应头来告知浏览器允许哪些源访问我） 
    - 告诉浏览器不需要跨域保护
    - 例子：
        cors({
    origin() {
        return "http://localhost:5173";
        },
    })
2. node 代理：绕开跨域代理
    - 同源策略位于浏览器上，只限制前端接收后端传输的数据，但是并不影响后端之间互相访问
    - 于是可以在前端定一个中转的后端，得到服务器端的数据,再传回前端，绕开了跨域问题

- 同源策略在浏览器上， 后端与后端之间访问不存在跨域
